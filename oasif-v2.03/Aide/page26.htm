Les propriétés du module : onglet Descriptif  

 

Titre du module : titre succinct de votre module, il sera affiché dans l’arborescence.

Objectif & Description du module : défini pour l’apprenant.
Cet objectif n ’est pas celui de l’enseignant ou de l ’institution.

Pré-requis demandés aux apprenants pour suivre de module.
les pré-requis peuvent être de diverses natures : connaissances préalables, méthode de travail, contraintes technologiques, etc...

Public visé : Public à qui s’adresse le module de formation.

Caractéristique du module : Bloquant (si l’apprenant ne réalise pas le module, il ne peut passer aux autres), à la carte (il offre plusieurs alternatives à l’apprenant sachant que l’un d’eux est obligatoire), optionnel (module non obligatoire).

Commanditaire / prescripteur : celui qui finance le module de formation s'il est différent de celui qui finance la formation.
Cette fonction est utile dans le cadre de conception de modules de formation spécialement adaptés pour un prescripteur (module sur mesure en formation continue par ex).

Porteur de la formation : celui qui le conçoit

Partenaires : les co-concepteurs

Client / bénéficiaire : c’est le destinataire final du module de formation

Nom du contact : nom de la personne contact pouvant fournir des renseignements complémentaires sur le module de formation.
Ces fonctions sont utiles dans le cadre de projet de formation regroupant plusieurs partenaires.

Crédits ECTS : nombre de crédits européen.

Nombre maximum d ’apprenants pouvant être accueillis simultanément dans une session du module.
Ce nombre est utilisé pour le calcul des temps d ’accompagnement défini dans les activités pédagogiques.

Lien "en savoir plus" : Lien vers un site (de type http://) contenant des informations complémentaires.

1er jour réel du module : Cette date est donnée et ne peut pas être modifiée ici. Elle dépend du premier jour réel de la formation et de la position du module sur le planning Formation.
